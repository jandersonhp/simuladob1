Questão 1

var x = "5"; var y = 3; alert(x + y);
  • a) 8
  • b) 53
  • c) 15
  • d) undefined
Correta: b) 53

Quando um dos operandos do + é string, o JS faz concatenação, não soma aritmética. "5" é string, então "5" + 3 vira "53".

Questão 2

Qual trecho sorteia um número de 1 a 6 (dado)?

var dado = __________ ; alert(dado);
  • a) Math.random() * 6
  • b) Math.floor(Math.random() * 6)
  • c) Math.floor(Math.random() * 6) + 1
  • d) Math.random(1, 6)
Correta: c) Math.floor(Math.random() * 6) + 1

Math.random() * 6 gera um decimal entre 0 e 5,999… (nunca chega no 6). O Math.floor() arredonda para baixo, resultando em 0, 1, 2, 3, 4 ou 5. Somando + 1, o resultado desloca para 1 até 6.

A opção b) daria de 0 a 5 e a opção a) daria número decimal (ex: 3,748).

Regra geral: Math.floor(Math.random() * N) gera de 0 até N−1. Para começar em 1, adicione + 1.

Questão 3

var a = 3; var b = a * 2; var c = a + b; alert(c);
  • a) 3
  • b) 6
  • c) 9
  • d) 12
Correta: c) 9

Execução sequencial: b = 3 × 2 = 6. Depois c = a + b = 3 + 6 = 9. O JavaScript executa linha por linha, de cima para baixo.

Questão 4

var nota = "7"; var passou = nota > 6; alert(passou);
  • a) true
  • b) false
  • c) undefined
  • d) Erro
Correta: a) true

Na comparação com > envolvendo string e número, o JavaScript converte a string "7" para número automaticamente (isso se chama coerção). Então 7 > 6 = true.

Questão 5

Qual trecho testa se o número está entre 10 e 20?

var n = parseInt(prompt("Número:")); if (______) { alert("Entre 10 e 20"); }
  • a) n > 10 e n < 20
  • b) n > 10 && n < 20
  • c) n > 10 || n < 20
  • d) 10 < n < 20
Correta: b) n > 10 && n < 20

No JavaScript, o operador E lógico é && e o OU é ||. A opção c) usa OU, o que faz a condição ser verdadeira para praticamente qualquer número. A opção d) com cadeia de comparação não funciona em JS como na matemática.

A opção a) usa "e" como texto, que é inválido como condicional.

Questão 6

var x = 10; var y = 3; alert(x % y);
  • a) 3
  • b) 1
  • c) 3,33
  • d) 0
Correta: b) 1

10 dividido por 3 é 3, e 3 × 3 = 9. Sobra 1. O operador % (módulo) retorna o resto da divisão inteira.

Pegadinha clássica de prova: muitos confundem módulo com divisão normal.

Questão 7

Qual a sequência de outputs?

var idade = 17; if (idade >= 18) { alert("A"); } else { alert("B"); } alert("Fim");
  • a) Primeiro "B", depois "Fim"
  • b) Só "Fim"
  • c) Primeiro "A", depois "Fim"
  • d) "B" (e para)
Correta: a) Primeiro "B", depois "Fim"

17 não é maior que 18, então cai no else → aparece "B". O alert("Fim") está fora do if/else, então ele executa sempre, independente da condição.

Dica de prova: código fora do bloco condicional SEMPRE executa depois.

Questão 8

O que formata o IMC com 2 casas decimais?

var imc = peso / (altura * altura); alert("Seu IMC: " + __________ );
  • a) imc.toFixed(2)
  • b) imc.round(2)
  • c) Math.floor(imc)
  • d) imc.casas(2)
Correta: a) imc.toFixed(2)

O método .toFixed(n) formata o número com n casas decimais e retorna uma string. Exemplo: (3.14159).toFixed(2) → "3,14".

Os métodos .round() e .casas() não existem em JavaScript — são pegadinhas comuns.

Questão 9 (pegadinha)

var a = true; var b = false; var c = true; if (a && b || c) { alert("Sim"); } else { alert("Não"); }
  • a) Sim
  • b) Não
  • c) undefined
  • d) Erro
Correta: a) Sim

No JavaScript, o operador && (E) tem precedência sobre o || (OU). Então a expressão é avaliada assim:

true && false = false
Depois: false || true = true

Resultado: entra no if → "Sim". Se não soubesse a precedência, poderia errar achando que o true && false anula tudo de cara.

Questão 10 — Escreva o código e rode

Escreva um programa que:

  1. Sorteia um número de 1 a 100
  2. Pede para o usuário digitar um palpite
  3. Se acertou → "Acertou!"
  4. Se palpite maior que sorteado → "Menor!"
  5. Se palpite menor que sorteado → "Maior!"

var sorteado = Math.floor(Math.random() * 100) + 1; var palpite = parseInt(prompt("Seu palpite (1-100):")); if (palpite == sorteado) { alert("Acertou!"); } else if (palpite > sorteado) { alert("Menor! O número era " + sorteado); } else { alert("Maior! O número era " + sorteado); }

Math.random() * 100 gera de 0 a 99,999… Math.floor() dá 0 a 99. Somando + 1 resulta em 1 a 100.

Atenção: sempre lembrar do + 1 quando a faixa precisa começar em 1, e não em 0!
